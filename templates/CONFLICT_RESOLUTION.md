## 两条硬约束（每块都必须核对）

- **约束①｜不回滚已有逻辑**：主干侧 + 分支侧的有效行为都保留。
- **约束②｜实现分支本次目标**：分支这次准备新增/修改的逻辑完整保留。
- 两者无法同时满足 → `NEEDS-HUMAN`，交人工，不擅自二选一。

## 冲突块对账（完整性证据，必须一致）

- git 报告冲突文件数：`<n>`（`git diff --name-only --diff-filter=U` 解决前快照）
- git 报告冲突块（hunk）总数：`<N>`（解决前统计，见下方命令）
- 本文档已记录条数：`<M>`（CR-* + CR-P-* 合计）
- **对账结论**：`N == M`？ ✅ 一致 / ❌ 不一致（不一致严禁提交，须补记）

> 解决前统计冲突块数：`grep -rc '^<<<<<<< ' .` 或 `git diff --diff-filter=U | grep -c '^<<<<<<<'`。
